## Data Pipeline — ESPN API v2

Fixture data is fetched from the **ESPN public API v2** (unofficial, no auth required).

### Key endpoints

```
# Scoreboard for a specific date
GET https://site.api.espn.com/apis/site/v2/sports/soccer/fifa.world/scoreboard
    ?dates=YYYYMMDD&lang=en&region=us

# Standings
GET https://site.api.espn.com/apis/v2/sports/soccer/fifa.world/standings
    ?lang=en&region=us
```

### League code

`fifa.world` — used for all FIFA World Cup 2026 requests.

### Fetch command

```bash
# Fetch Round of 32 (default)
uv run python manage.py fetch_fixtures

# Fetch a specific stage
uv run python manage.py fetch_fixtures --stage round-of-16

# Fetch all knockout stages at once
uv run python manage.py fetch_fixtures --stage all
```

Available `--stage` values: `round-of-32`, `round-of-16`, `quarterfinals`,
`semifinals`, `third-place`, `final`, `all`.

Output lands in `trivia/data/<stage>.json`.

## FootballMatch Schema

Each entry in `matches[]` follows this shape (mirrors the TypeScript
`FootballMatch` / `TeamScore` types):

```jsonc
{
  "id": "760486",
  "leagueCode": "fifa.world",
  "leagueName": "FIFA World Cup",
  "date": "2026-06-28T19:00Z",        // UTC ISO-8601
  "stage": "Round of 32",
  "status": "scheduled",              // "scheduled" | "in progress" | "full time" | …
  "state": "pre",                     // "pre" | "in" | "post"
  "minute": "0'",
  "detail": "Sun, June 28th at 3:00 PM EDT",
  "clockSeconds": 0,
  "clockUpdatedAt": 1750000000000,    // Unix ms
  "clockRunning": false,
  "inPenalties": false,
  "venue": {
    "name": "SoFi Stadium",
    "city": "Inglewood, California",
    "country": "USA"
  },
  "home": {
    "id": "5926",
    "name": "Group A 2nd Place",      // Real name once groups finish
    "abbreviation": "2A",
    "score": 0,
    "logo": "https://a.espncdn.com/…"
  },
  "away": { /* same shape */ }
}
```

### Group position codes (pre-knockout)

| Code    | Meaning                                      |
|---------|----------------------------------------------|
| `1X`    | Winner (1st place) of Group X                |
| `2X`    | Runner-up (2nd place) of Group X             |
| `3RD`   | Best third-place team (bracket-slot specific)|
| `GER`   | Germany — already confirmed group winner     |
| `MEX`   | Mexico — already confirmed group winner      |
| `USA`   | United States — already confirmed group winner|

## Settings Notes

- `DJANGO_SETTINGS_MODULE` = `config.settings` (was `pets.settings` in older commits)
- All times in the JSON are **UTC**; convert to local (Guatemala = UTC-6) in the frontend
- `DEBUG = False` in settings; set `SECRET_KEY` and DB env vars via `.env`

## Do / Don't for agents

**Do:**
- Run `uv run python manage.py fetch_fixtures` to refresh fixture data
- Keep all code and JSON **in English**
- Use `uv run` prefix for all Python commands (pyenv version may not be installed globally)
- Add new stages to `STAGE_DATE_RANGES` in `fetch_fixtures.py` when advancing rounds

**Don't:**
- Don't add Tailwind — the project uses CSS
- Don't modify `dieciseisavos.json` manually; it is superseded by `round-of-32.json`
